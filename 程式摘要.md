崇華十六期整合管理系統 - 技術摘要與維護指南 (2026 最終整合版)
本文件旨在為系統開發者提供深層的架構說明與技術邏輯，確保系統長期的穩定性與可維護性。

## 1. 核心系統架構 (Serverless Design)
本系統採用全雲端 Serverless 架構，極大化降低伺服器維護成本與門檻：
- **前端 (Frontend)**: 單一 HTML 檔案 (`index.html`)，整合 Tailwind CSS (UI)、Vanilla JS (邏輯)、Chart.js (數據視覺化)。
- **後端 (Backend)**: Google Apps Script (GAS)，以 Web App 形式發布，處理所有 CRUD (增刪改查) 請求。
- **資料庫 (Database)**: Google Sheets (試算表)。利用工作表 (Sheet) 作為關聯或日誌型儲存媒介。
- **外部整合**: 
  - **圖檔儲存**：整合 GitHub API。
  - **雲端備份**：透過 Google Drive API 上傳至指定資料夾（可透過 `Config` 工作表動態修改 `DRIVE_FOLDER_ID`）。

## 2. 資料存取邏輯 (Data Access Flow)
- **通訊機制**: 統一使用 `fetch(APPS_SCRIPT_URL, { method: 'POST', ... })`。
- **動態名單獲取**: 前端由 `getTeamsData` Action 向 GAS 獲取最新學員名冊，自動適應 Users 工作表的變動。
- **跨國組別支援 [2026/04 新增]**: 系統已全面支持「馬來西亞」組別，後端統計與前端圖表顏色均已同步更新，確保跨區域運行的數據正確性。
- **角色權限 (Role-Based Control)**: 
  - 系統採用**不區分大小寫**的角色判定。符合 `Admin`、`Teacher`、`Leader` 等關鍵字即可取得對應權限。
  - 管理員與老師可查看全區報告，組員則僅限於個人歷史紀錄。

## 3. 特色功能實作細節

### A. 強化版作業繳交追蹤系統 [2026/04 重大升級]
- **點名報告聯動 (Attendance Linkage)**: 
  - 實作「預設隱藏」邏輯，優化長列表效能與可讀性。
  - 支援「查看所有組別」功能，並自動統計組別人數。
  - **個人雙軌積分顯示**: 在組員名單中，所有成員姓名下方會並排渲染「🏆 當表加分」與「🌟 全期總共」，清楚呈現個人在特定表與歷來的得分狀態。
- **管理員透視功能 (Admin Overview)**:
  - 管理員點擊成員旁的「文件圖示」即可彈出 `member-hw-modal`，隨時查閱該學員的完整歷史作業清單。
- **作業報告聚合 (Report Aggregation)**:
  - 修正了原本重複顯示的問題。現在 `renderHomeworkReport` 會將同主題、同姓名的多筆繳交紀錄合併。
  - 卡片會顯示該主題的「總繳交次數」與「最新繳交時間」，點擊卡片即可展開具體的心得內容與附件。

### B. 加分與統計圖表 (Bonus System)
- **多區域統計**: 支持第一至第七組及馬來西亞組，圖表顏色採用高辨識度配色（馬來西亞固定為橘色 `#fb923c`）。
- **雙軌數據展示**: 同時呈現「單次加分表累計」與「全期總累計」，並支援前端選單即時切換不同的加分表單（加分_YYYY/MM/DD）。

### C. 測驗解析系統 (Quiz Engine)
- **強健解析**: 支援自動識別題號、題目圖片轉直連連結、複選題多重答案比對。
- **成績保存**: 結果存儲於「結果YYYY/MM」分頁，支援自動擴充新題目的欄位表頭。

### D. 外部報表整合 (External Reports) [2026/05 新增]
- **區域出席統計聯動**: 
  - 系統支援即時從另一個獨立的 Google Spreadsheet 動態讀取「區域」工作表的資料（限制範圍 `F1:O84`）。
  - 將資料動態轉換為無縫 HTML 表格，並提供「重整」按鈕隨時更新數據。
- **無縫圖片與 PDF 匯出**:
  - 整合 `html2canvas` 與 `jsPDF` 套件，允許使用者將動態生成的報表匯出為高品質 JPG 或單頁長條形 PDF。
  - PDF 匯出邏輯特別設定為依照報表內容動態計算頁面長寬，確保所有資料在單一頁面中無縫銜接，不會因 A4 分頁而被裁切。

## 4. UI/UX 與交互優化
- **多層彈窗層級解決 (Z-Index Overlap)**:
  - 為了解決「視窗中的視窗」遮擋問題，系統將「詳細內容（測驗詳解、作業心得）」的 `z-index` 提升至 2500。
  - 基礎清單視窗（如作業列表）保持在 2000，確保層級分明。
- **響應式與視覺設計**:
  - 採用 Glassmorphism (玻璃擬態) 設計，搭配細緻的微動畫與陰影效果。
  - 全面優化手機端操作，所有點擊區域均符合人體工學。

## 5. 維護流程 (Maintenance Flow)
- **日誌追蹤 (Action Log)**: 每一筆加分、簽到、作業繳交均會記錄操作人 (Operator) 與精確時間。
- **部署規範**: 在 GAS 部署新版本後，必須將新的 Web App URL 更新至 `index.html` 的 `APPS_SCRIPT_URL`。
- **工作表命名規範**: 
  - `Users`: 成員清單。
  - `加分_...`: 積分紀錄。
  - `作業_...`: 作業提交內容。
  - `簽到YYYY/MM/DD`: 每日出席。
  - `結果YYYY/MM`: 測驗成績。
