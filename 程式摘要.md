崇華十六期整合管理系統 - 技術摘要與維護指南 (詳盡版)
本文件旨在為系統開發者提供深層的架構說明與技術邏輯，確保系統長期的穩定性與可維護性。

1. 核心系統架構 (Serverless Design)
本系統採用全雲端 Serverless 架構，極大化降低伺服器維護成本與門檻：
- **前端 (Frontend)**: 單一 HTML 檔案 (`index.html`)，整合 Tailwind CSS (UI)、Vanilla JS (邏輯)、Chart.js (數據視覺化)。
- **後端 (Backend)**: Google Apps Script (GAS)，以 Web App 形式發布，處理所有 CRUD (增刪改查) 請求。
- **資料庫 (Database)**: Google Sheets (試算表)。利用工作表 (Sheet) 作為關聯或日誌型儲存媒介。
- **外部整合**: 
  - 圖檔儲存：整合 GitHub API。
  - 雲端備份：透過 Google Drive API 上傳至指定資料夾（可透過 `Config` 工作表動態修改 `DRIVE_FOLDER_ID`）。

2. 資料存取邏輯 (Data Access Flow)
- **通訊機制**: 統一使用 `fetch(APPS_SCRIPT_URL, { method: 'POST', ... })`。部分唯讀資料採用 `GET` 請求。
- **組別獲取 (getTeamsData) [新]**: 
  - 前端改由 `getTeamsData` Action 向 GAS 獲取最新學員名冊，解決過往依賴靜態 GID 導致的維護困難。
- **動態表頭 (Dynamic Headers)**: 
  - 在 `handleSubmitHomework` 與 `handleSubmitQuiz` 中，後端會自動比對傳入 JSON 中的 Question 與現有工作表首行，若出現新題目則自動使用 `setValues` 擴充欄位，無需手動修改資料表。
- **角色權限 (Role-Based Control)**: 
  - 系統依據 `Users` 工作表中的 `role` 欄位決定權限。管理員 (Admin/Teacher) 可查看全區報告，組長 (Leader) 僅能看到該組數據。

4. 特色功能實作細節
- **加分與統計圖表**:
  - 後端專屬 Action: `getBonusStats` 會讀取最新的「加分_」分頁，按組別 (Group) 進行分數聚合。
  - 前端 Chart.js: 接收聚合後的 JSON 並渲染成直條圖。實作 `bonusChartInstance.destroy()` 機制防止圖表重疊渲染錯誤。
- **點名報告優化**: 
  - 實作「預設隱藏」邏輯，管理員需先選取組別或「所有組別」才會渲染名單。
  - **累計加分顯示**: 點名報告名單中，會自動顯示每位學員在當前加分表中的累計總得分 (`pts`)。
  - **全方位名冊 & UI 精簡 [更新]**: 點名報告會顯示所有組員，但「未簽到者」僅保留狀態圓點，隱藏其狀態文字（如 "null" 或 "未簽到"），使畫面專注於已簽到者的時間紀錄與累計加分。

5. 維護流程 (Maintenance Flow)
- **日誌追蹤與人員追溯 (Action Log) [更新]**: 
  - 系統會記錄所有關鍵操作（登入、簽到、註冊、加分等）至 `Log` 工作表。
  - **新增操作人員欄位**: Log 現在包含「操作人員 (Operator)」欄位，明確追蹤是哪位管理員進行了加分或代理簽到，便於問題回溯與責任釐清。
- **權限分級優化**: 
  - 「建立新加分頁」功能限制為僅有管理員 (`Admin`) 可使用。
- **更新後端**: 在 GAS 編輯器存檔後，必須執行「部署」->「新部署」，並將產生的 URL 更新至 `index.html` 的 `APPS_SCRIPT_URL`。
- **下拉選單點選修復**: 修正了 SVG 圖示遮擋 `select` 點選的問題。
- **試算表保護**: 請勿更動 `Config`、`Users`、`HomeworkSettings`、`Locations` 與 `Log` 的名稱。
