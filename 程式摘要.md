崇華十六期整合管理系統 - 技術摘要與維護指南 (詳盡版)
本文件旨在為系統開發者提供深層的架構說明與技術邏輯，確保系統長期的穩定性與可維護性。

1. 核心系統架構 (Serverless Design)
本系統採用全雲端 Serverless 架構，極大化降低伺服器維護成本與門檻：
- **前端 (Frontend)**: 單一 HTML 檔案 (`index.html`)，整合 Tailwind CSS (UI)、Vanilla JS (邏輯)、Chart.js (數據視覺化)。
- **後端 (Backend)**: Google Apps Script (GAS)，以 Web App 形式發布，處理所有 CRUD (增刪改查) 請求。
- **資料庫 (Database)**: Google Sheets (試算表)。利用工作表 (Sheet) 作為關聯或日誌型儲存媒介。
- **外部整合**: 
  - 圖檔儲存：整合 GitHub API。
  - 雲端備份：透過 Google Drive API 上傳至指定資料夾（可透過 `Config` 工作表動態修改 `DRIVE_FOLDER_ID`）。

2. 資料存取邏輯 (Data Access Flow)
- **通訊機制**: 統一使用 `fetch(APPS_SCRIPT_URL, { method: 'POST', ... })`。部分唯讀資料採用 `GET` 請求（如透過 Visualization API 讀取特定 GID）。
- **組別獲取 (getTeamsData)**: 前端改由 `getTeamsData` Action 向 GAS 獲取最新學員名冊，解決過往依賴靜態 GID 導致的維護困難。
- **動態表頭 (Dynamic Headers)**: 在 `handleSubmitHomework` 與 `handleSubmitQuiz` 中，後端會自動比對傳入 JSON 中的 Question 與現有工作表首行，若出現新題目則自動使用 `setValues` 擴充欄位。
- **角色權限 (Role-Based Control) [更新]**: 
  - 系統全面採用**不區分大小寫**的角色判定邏輯。不論資料庫標記為 `Admin` 或 `ADMIN`，只要符合關鍵字即可取得完整管理權限。
  - 管理員 (Admin/Teacher) 可查看全區報告，組長 (Leader) 僅能看到該組數據。

3. 特色功能實作細節
- **測驗解析系統 (Quiz Engine) [升級]**:
  - **強健解析**: 支援中文題型（單選、複選、填充），並能自動過濾空白行與識別智慧型題號。
  - **靈活分割**: 支援全/半形逗號、分號、換行等不同格式的選項與答案分割。
  - **直連圖片**: `convertToDirectLink` 支援多種 Google Drive 連結格式與 ID，自動偵測並轉為預覽直連。
- **加分與統計圖表**:
  - **圖表標題**: 統一標註為「崇華班第16期小組積分表」。
  - **雙長條圖展示**: 整合最新數據與總累積數據。
- **點名報告與 UI**:
  - 實作「預設隱藏」邏輯，優化長列表效能與可讀性。
  - 支援「查看所有組別」功能，並自動統計組別人數。

4. 維護流程 (Maintenance Flow)
- **日誌追蹤 (Action Log)**: 記錄包含「操作人員 (Operator)」在內的所有關鍵操作。
- **權限分級**: 建立加分頁等高敏操作僅限管理員級別。
- **部署方式**: 在 GAS 部署新版本後，必須將新的 Web App URL 更新至 `index.html`。
- **試算表保護**: 請勿更動核心工作表（Config, Users, HomeworkSettings, Locations, Log）的名稱。
